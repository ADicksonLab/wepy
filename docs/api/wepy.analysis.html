spanning_contig_traces()[source]¶

Returns a list of all possible spanning contigs given the continuations present in this file. Contigs are a list of runs in the order that makes a continuous set of data. Spanning contigs are always as long as possible, thus all must start from a root and end at a leaf node.

This algorithm always returns them in a canonical order (as long as the runs are not rearranged after being added). This means that the indices here are the indices of the contigs.

Contigs can in general are any such path drawn from what we call the “contig tree” which is the tree (or forest of trees) generated by the directed edges of the ‘continuations’. They needn’t be spanning from root to leaf.
