21class TaskMapper(ABCWorkerMapper): 

22 """Process-per-task mapper. 

23 

24 This method of work mapper starts new processes for each runner 

25 segment task that needs to be run. This allows cheap copying of 

26 shared state using the operating system primitives. On linux this 

27 would be either 'fork' (default) or 'spawn'. Fork is cheap but 

28 doesn't initialize certain process namespace things, whereas spawn 

29 is much more expensive but properly cleans things up. Fork should 

30 be sufficient in most cases, however spawn may be needed when you 

31 have some special contexts in the parent process. This is the case 

32 with starting CUDA contexts in the main parent process and then 

33 forking new processes from it. We suggest using fork and avoiding 

34 making these kinds of contexts in the main process. 

35 

36 This method avoids using shared memory or sending objects through 

37 interprocess communication (that has a serialization and 

38 deserialization cost associated with them) by using OS copying 

39 mechanism. However, a new process will be created each cycle for 

40 each walker in the simulation. So if you want a large number of 

41 walkers you may experience a large overhead. If your walker states 

42 are very small or a very fast serializer is available you may also 

43 not benefit from full process address space copies. Instead the 

44 WorkerMapper may be better suited. 

45 

46 """
